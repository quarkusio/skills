# Migration File Organization Guide

## Overview

This document defines the standard file organization structure for Spring to Quarkus migrations. All agents MUST follow this structure to ensure consistency and ease of navigation.

## Directory Structure

```
<quarkus_target_dir>/
├── migration-reports/              # All phase-specific migration reports
│   ├── phase-03-project-bootstrap.json
│   ├── phase-04-database-migration.json
│   ├── phase-05-persistence-migration.json
│   ├── phase-06-service-migration.json
│   ├── phase-07-messaging-migration.json
│   ├── phase-08-web-migration.json
│   ├── phase-08b-web-views-migration.json
│   ├── phase-09-configuration-migration.json
│   ├── compile-fix-report.json
│   └── phase-11-validation.json
│
├── migration-metadata/             # Discovery, planning, and tracking metadata
│   ├── repo-metadata.json
│   ├── dependency-analysis.yaml
│   ├── migration-context.json
│   ├── code-metadata.yaml
│   ├── spring-messaging.json
│   ├── quarkus-messaging.json
│   ├── spring-config-metadata.json
│   └── quarkus-config-metadata.json
│
├── migration-spec.yaml             # PRIMARY: Migration specification (root level)
├── migration-summary.md            # PRIMARY: Final migration summary (root level)
├── pom.xml                        # Quarkus project POM
│
└── src/                           # Application source code
    ├── main/
    │   ├── java/
    │   └── resources/
    │       ├── application.properties
    │       └── import.sql (if applicable)
    └── test/

<spring_source_dir>/
├── migration-metadata/             # Source project metadata
│   ├── repo-metadata.json          # Generated by discovery agent
│   ├── dependency-analysis.yaml    # Generated by dependency analysis agent
│   └── code-metadata.yaml          # Generated for validation
│
└── [original Spring project structure]
```

## File Categories and Locations

### 1. Primary Migration Artifacts (Root Level)
**Location**: `<quarkus_target_dir>/`

These are the most important files users will reference:

- **migration-spec.yaml** - The binding contract between all agents, contains all migration decisions
- **migration-summary.md** - Human-readable final report with statistics and recommendations

**Rationale**: Root level for maximum visibility and easy access

### 2. Phase Reports (migration-reports/)
**Location**: `<quarkus_target_dir>/migration-reports/`

All phase-specific JSON reports documenting what was migrated:

- **phase-03-project-bootstrap.json** - Project setup details (Phase 3)
- **phase-04-database-migration.json** - Database schema and config migration (Phase 4)
- **phase-05-persistence-migration.json** - Entity and repository migration (Phase 5)
- **phase-06-service-migration.json** - Service layer migration (Phase 6)
- **phase-07-messaging-migration.json** - Messaging migration (Phase 7, if applicable)
- **phase-08-web-migration.json** - REST controller migration (Phase 8)
- **phase-08b-web-views-migration.json** - Web views migration (Phase 8b, if applicable)
- **phase-09-configuration-migration.json** - Configuration migration (Phase 9)
- **compile-fix-report.json** - Compilation fixes (Phase 10, as needed)
- **phase-11-validation.json** - Final validation results (Phase 11)

**Rationale**: Grouped together for easy review of migration details per phase

### 3. Metadata Files (migration-metadata/)
**Location**: 
- Source: `<spring_source_dir>/migration-metadata/`
- Target: `<quarkus_target_dir>/migration-metadata/`

Discovery, planning, and tracking files:

- **repo-metadata.json** - Source repository analysis (Phase 1, source dir)
- **dependency-analysis.yaml** - Dependency mapping (Phase 1b, source dir)
- **migration-context.json** - Progress tracking (Orchestrator, target dir)
- **code-metadata.yaml** - Code structure for validation (Validators, both dirs)
- **spring-messaging.json** - Spring messaging metadata (Phase 7 validator, source dir)
- **quarkus-messaging.json** - Quarkus messaging metadata (Phase 7 validator, target dir)
- **spring-config-metadata.json** - Spring configuration metadata (Phase 9 validator, source dir)
- **quarkus-config-metadata.json** - Quarkus configuration metadata (Phase 9 validator, target dir)

**Rationale**: Separates technical metadata from user-facing reports

### 4. Application Files (Standard Locations)
**Location**: `<quarkus_target_dir>/` (root) and `src/`

Standard Quarkus project structure (created by Phase 3 and subsequent phases):
- **pom.xml** - Maven configuration (Phase 3)
- **src/main/java/** - Application code (Phases 5-8)
- **src/main/resources/application.properties** - Configuration (Phase 9)
- **src/main/resources/import.sql** - Database initialization (Phase 4, if applicable)
- **src/test/** - Test code (migrated by various phases)

**Note**: Additional files like Dockerfile, docker-compose.yml, README.md may be created by Phase 9 (Configuration) depending on the migration spec.

**Rationale**: Standard Quarkus conventions

## Agent Responsibilities

### Discovery Agent (Phase 1)
**Outputs**:
- `<spring_source_dir>/migration-metadata/repo-metadata.json`

### Dependency Analysis Agent (Phase 1b)
**Outputs**:
- `<spring_source_dir>/migration-metadata/dependency-analysis.yaml`

### Planning Agent (Phase 2)
**Outputs**:
- `<quarkus_target_dir>/migration-spec.yaml` (root level)

### Project Bootstrap Agent (Phase 3)
**Outputs**:
- `<quarkus_target_dir>/pom.xml`
- `<quarkus_target_dir>/src/main/resources/application.properties`
- `<quarkus_target_dir>/migration-reports/phase-03-project-bootstrap.json`

### Database Migration Agent (Phase 4)
**Outputs**:
- `<quarkus_target_dir>/src/main/resources/import.sql`
- `<quarkus_target_dir>/migration-reports/phase-04-database-migration.json`

### Persistence Migration Agent (Phase 5)
**Outputs**:
- `<quarkus_target_dir>/src/main/java/**/*Entity.java`
- `<quarkus_target_dir>/src/main/java/**/*Repository.java`
- `<quarkus_target_dir>/migration-reports/phase-05-persistence-migration.json`

### Service Migration Agent (Phase 6)
**Outputs**:
- `<quarkus_target_dir>/src/main/java/**/*Service.java`
- `<quarkus_target_dir>/migration-reports/phase-06-service-migration.json`

### Messaging Migration Agent (Phase 7)
**Outputs**:
- `<quarkus_target_dir>/src/main/java/**/*Processor.java`
- `<quarkus_target_dir>/migration-reports/phase-07-messaging-migration.json`

### Web Layer Migration Agent (Phase 8)
**Outputs**:
- `<quarkus_target_dir>/src/main/java/**/*Resource.java`
- `<quarkus_target_dir>/migration-reports/phase-08-web-migration.json`

### Configuration Migration Agent (Phase 9)
**Outputs**:
- `<quarkus_target_dir>/Dockerfile`
- `<quarkus_target_dir>/docker-compose.yml`
- `<quarkus_target_dir>/Makefile`
- `<quarkus_target_dir>/README.md`
- `<quarkus_target_dir>/migration-reports/phase-09-configuration-migration.json`

### Validation Agent (Phase 11)
**Outputs**:
- `<quarkus_target_dir>/migration-reports/phase-11-validation.json`
- `<quarkus_target_dir>/migration-metadata/testlogs.txt`

### Reporting Agent (Final)
**Outputs**:
- `<quarkus_target_dir>/migration-summary.md` (root level)

## Validation Metadata

Validators generate code metadata files for comparison:

**Source Project**:
```bash
# Build validator if needed
cd validators/java && mvn clean package -DskipTests -q

java -jar validators/java/target/migration-validator-1.0.0.jar extract metadata \
  <spring_source_dir> -o <spring_source_dir>/migration-metadata/code-metadata.yaml
```

**Target Project**:
```bash
java -jar validators/java/target/migration-validator-1.0.0.jar extract metadata \
  <quarkus_target_dir> -o <quarkus_target_dir>/migration-metadata/code-metadata.yaml
```

## Migration Context Tracking

The orchestrator maintains `<quarkus_target_dir>/migration-metadata/migration-context.json` with:

```json
{
  "generatedAt": "2024-01-15T10:30:00Z",
  "sourceRepo": "/path/to/spring/project",
  "targetRepo": "/path/to/quarkus/project",
  "currentPhase": "phase-06-service",
  "completedPhases": ["phase-01-discovery", "phase-02-planning", ...],
  "approvedPhases": ["phase-01-discovery", "phase-02-planning", ...],
  "paths": {
    "repoMetadata": "migration-metadata/repo-metadata.json",
    "dependencyAnalysis": "migration-metadata/dependency-analysis.yaml",
    "migrationSpec": "migration-spec.yaml"
  },
  "phaseReports": {
    "phase-03-bootstrap": "migration-reports/phase-03-project-bootstrap.json",
    "phase-04-database": "migration-reports/phase-04-database-migration.json",
    ...
  }
}
```

## Benefits of This Structure

1. **Clear Separation**: Reports, metadata, and application code are clearly separated
2. **Easy Navigation**: Users can quickly find migration reports in one location
3. **Standard Compliance**: Follows Quarkus project conventions
4. **Version Control Friendly**: Metadata can be easily .gitignored if desired
5. **Audit Trail**: Complete history of migration decisions and outcomes
6. **Resumability**: migration-context.json enables resuming interrupted migrations

## .gitignore Recommendations

Add to `<quarkus_target_dir>/.gitignore`:

```gitignore
# Migration metadata (optional - keep for audit trail)
migration-metadata/
migration-reports/

# Keep these for reference
!migration-spec.yaml
!migration-summary.md
```

## Implementation Notes

- All agents MUST create the `migration-reports/` and `migration-metadata/` directories if they don't exist
- File paths in migration-spec.yaml and migration-context.json MUST be relative to the target project root
- Backup files (.bak) MUST go in `migration-metadata/backups/`
- Test logs and temporary files MUST go in `migration-metadata/`
